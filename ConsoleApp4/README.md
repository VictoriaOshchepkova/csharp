# Алгоритм работы

## Задание №1.8. List

Описание:  
Составить программу, которая вставляет в список L за первым вхождением элемента E все элементы списка L, если E входит в L. 

Алгоритм:

1. Метод для решения задачи принимает на вход список для вставки, элемент, после которого происходит вставка, и вставляемый список
2. Проверяется наличие элемента в первом списке. Если отсутсвтует - возвращается исключение
3. Иначе второй список вставляется в первый после первого вхождения элемента

## Задание №2.8 LinkedList

Описание:  
Добавить в начало и конец списка L новый элемент E. 

Алгоритм:

1. Метод для решения задачи принимает на вход список для вставки и вставляемый элемент
2. Элемент вставляется в начало и конец списка

## Задание №3.8 Hashset

Описание:  
Есть перечень названий книг. Определить для каждой книг, какие из них прочли все из n читателей, какие — некоторые из читателей, и какие — никто из читателей. 

Алгоритм:

1. Метод для решения задачи принимает на вход множество книг B и множество читателей R, где читатель - это множество RB прочитанных им книг 
2. Находится множество книг, прочитанных всеми, с помощью поэтапного пересечения множества B со всеми множествами RB
3. Находится множество книг, прочитанных хоть кем-то, с помощью поэтапного объединения всех множеств RB
4. Находится множество книг, прочитанных никем, с помощью разности множества всех книг и множества книг, прочитанных хоть кем-то
   
## Задание №4.8 Hashset

Описание:  
Файл содержит текст на русском языке. Напечатать те символы, которых нет в первом слове, но они присутствуют в каждом из других. 

Алгоритм:

1. Метод для решения задачи принимает на вход путь к файлу, где хранится текст
2. Текст из файла считывает в строковую переменную
3. Создается массив всех слов в тексте с помощью разделения текста по основным разделителям (пробел, точка, запятая и т.д.)
4. Создается множество F всех букв в первом слове
5. Создается множество слов O, где слово - это множество букв OC, содержащихся в нем 
6. Находится множество всех букв, содержащихся во всех словах, кроме первого, с помощью поэтапно пересечения множеств OC, а затем разности получившегося множества со множеством F

## Задание №5.8 Dictionary/SortedList 

Описание:  
Школьная олимпиада по информатике проводилась для учеников 7-11-хклассов, участвующих в 
общем конкурсе. Каждый участник олимпиады мог набрать от 0 до 70 баллов. Для определения 
призеров олимпиады сначала отбираются 25% участников, показавших лучшие результаты. 
Если у последнего участника, входящего в 25%,оказывается такое же количество баллов, как и у 
следующих за ним в итоговой таблице, все они считаются призерами только тогда, когда 
набранные ими баллы больше половины максимально возможных; иначе все они не считаются 
призерами. 
Напишите эффективную по времени работы и по используемой памяти программу, которая по 
результатам олимпиады будет определять минимальный балл призера олимпиады, и 
количество призеров было в каждой параллели (среди 7-х, 8-х, 9-х, 10-х и 11-х классов отдельно). 
Гарантируется, что хотя бы одного призера по указанным правилам определить можно.  
На вход программе сначала подается число участников олимпиады N. В каждой из следующих N 
строк находится результат одного из участников олимпиады в следующем формате: 
<Фамилия><Имя><класс><баллы> 
где <Фамилия> – строка, состоящая не более, чем из 30 символов,<Имя> – строка, состоящая не 
более, чем из 15 символов, <класс> – число от 7 до 11, <баллы> – целое число от 0 до 70 
набранных участником баллов. <Фамилия> и <Имя>, <Имя> и <класс>, а также <класс> и<баллы> 
разделены одним пробелом. Пример входной строки: 
Семенов Сидор 11 66 
Программа должна выводить в первой строке минимальный балл призера, а в следующей – 
число призеров по всем параллелям отдельно. 
Пример выходных данных: \
63 \
1 5 8 12 22 

Алгоритм:

1. Метод для решения задачи принимает на вход путь к файлу, где хранятся данные об участниках олимпиады
2. Данные считываются в массив, количество записей считывается в целочисленную переменную
3. Создается SortedList SL, где ключ - балл, значение - количество участников, получивших этот балл
4. Создается словарь D, где ключ - класс, значение - SortedList DSL, где ключ - балл, значение - количество участников, получивших этот балл и учащихся в этом классе
5. Данные из массива записей считываются и записываются в SortedList SL и словарь D
6. Рассчитывается предполагаемое количество призеров T - 25% от всех участвующих
7. Проходится по SL в порядке убывания баллов и суммируется количество участников, входящих в T, и которые заработали больше 35 баллов. Текущий балл фиксируется
8. Как только накопленная сумма достигает T - фиксируется минимальный балл и цикл завершается
9. Если накопленная сумма превышает T - цикл завершается
10. По завершению цикла находится минимальный балл призера олимпиады
11. Если не нашелся ни один участник с >35 баллами, выводится соответствующее оповещение
12. Иначе проходится по DSL и считывает количество призеров в каждом классе

Пройти коллекцию в обратном порядке (от большего к меньшему)

Накопительно считать количество элементов, пока не достигнут лимит

При этом учитывать дополнительное условие для значений элементов

Запомнить минимальное значение, которое вошло в отбор

Программа состоит из 2 файлов:

# Алгоритм работы

## Задание №6.8. Перегрузка операций

Описание:  
Создать класс Money с полями uint rubles и byte kopeks. Создать метод: Вычитание переменной типа Money (учесть, что денежная величина не может быть меньше 0). Результат должен быть типа Money. 

Алгоритм:

1. Конструктор по умолччанию присваивает полям значения 0
2. Конструктор с параметрами нормализует передаваемые значения с помощью дополнительного метода (если копеек больше 99 - перебрасывает в рубли)
3. Метод принимает на вход объект класса Money
   - Переводит все данные исходного Money в копейки
   - Переводит все данные передаваемого Money в копейки
   - Находит их разность. Если результат меньше 0 - создается исключение
   - Создается новый объект Money с вычисленными данными
4. Перегруженный метод ToString() возращает строку в формате "{Rubles} руб. {Kopeks} коп."

## Задание №7.8 Перегрузка операций

Описание:  
Перегрузить операции для класса Money:\
Унарные операции: \
++ добавление копейки к объекту типа Money \
-- вычитание копейки из объекта типа Money \
Операции приведения типа: \
uint (неявная) – результатом является количество рублей (копейки отбрасываются); \
double(явная) – результатом является копейки, переведенные в рубли, целые рубли отбрасываются, результат <1. \
Бинарные операции:\
– Money m, беззнаковое целое число (лево- и правосторонние операции), результат должен быть типа Money.\
– Money m, Money m, результат должен быть типа Money.\
Результат не может быть отрицательным. 

Алгоритм:

++: к значению копеек прибавляется 1, при надобности нормализуется (если копеек больше 99 - перебрасывает в рубли)\
--: тот же принцип, что в п. 1., с доп. проверками на нулевые значения (исключение при попытке вычесть из 0)\
uint (неявная): возвращается значение поля с рублями\
double (явная): возвращается значение поля с копейками, деленное на 100\
– (Money m, Money m): возвращается значение из метода из Задания №6.8. п. 3.\
– (Money m, беззнаковое целое число): создается новый объект Money со значением поля с рублями, равным безнаковому целому числу, и возвращается значение разности с помощью перегруженной операции из п. 5.
